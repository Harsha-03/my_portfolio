# UX Design Process — Harsha Asapu

## Overview

Harsha follows a user-centered design process grounded in clarity, iteration,
and real-world constraints. His process is shaped by working on both client-driven
and self-directed products, with a consistent emphasis on shipped, verifiable outcomes.

---

## Core Process

### 1. Discovery & Problem Framing

- Understand the user, their context, and the constraints before designing anything
- Identify the real problem vs the stated problem
- Align with stakeholders on goals and success criteria
- Tools: conversations, requirement mapping, success criteria definition

### 2. Information Architecture

- Structure content and navigation before visual design
- Map user flows: what does the user need to do, in what order, with what context
- Single-page vs multi-page decisions based on user behavior, not convention
- Reduce cognitive load through hierarchy and grouping

### 3. Wireframing & Flow Design

- Low-fidelity wireframes to test structure before investing in visuals
- Iterate on layout, hierarchy, and flow with stakeholders before high-fidelity
- Tools: Figma (Auto Layout, Components, Responsive frames)

### 4. High-Fidelity Prototyping

- Build interactive, production-close prototypes in Figma
- Responsive layouts tested across breakpoints
- Interaction patterns defined for developer handoff
- Prototype fidelity aligned with what the dev team needs to implement accurately

### 5. Usability Testing & Iteration

- Test with real users — minimum 10–20 for meaningful vulnerability discovery
- Document patterns, blockers, and friction points
- Iterate based on observed behavior, not assumptions
- Has tested his own RAG portfolio assistant with 10–20 users to find response gaps

### 6. Handoff & Collaboration

- Works in Agile environments with Slack-based communication
- Aligns with product managers and developers throughout (not at the end)
- Uses Jira for task tracking, Notion for documentation

---

## Design Principles Harsha Applies

- **Comprehension over impression:** A design that users understand quickly is more
  valuable than one that impresses visually
- **Restraint over decoration:** Remove anything that competes with the content
- **Behavioral design:** Design for how people actually behave, not how they should
- **Shipped beats perfect:** Real products with known trade-offs beat unshipped concepts
- **Trust through verification:** Interfaces should help users verify claims, not just read them

## Tools

- Figma: Auto Layout, Components, Variants, Interactive Prototyping, Responsive Design
- Design Systems: building and maintaining component libraries
- Jira: Agile task management
- Notion: design documentation
- Git/GitHub: version control for frontend implementation
- Framer: motion and interaction exploration

---

## Frontend Awareness

Harsha builds what he designs. He implements in React, Next.js, and Tailwind CSS,
which means his design decisions are grounded in what is technically feasible.
He understands how designs translate into code, avoiding designs that look good in
Figma but are impractical to ship.
